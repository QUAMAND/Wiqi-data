# 문서 작성을 위한 가이드

이 문서는 일관된 형식과 가독성을 유지하기 위한 `.md` 문서 작성 규칙을 설명합니다.

- 해당 위키는 `react`와 `rehypeRaw` 그리고 `ReactMarkdown`으로 `.md` 파일을 불러옵니다.

---

# 1. 제목 구조
문서의 구조는 `#`을 사용해 계층적으로 작성합니다.

> ```js
> # 제목
> ## 부제목
> ### 소제목
> #### 세부 항목
> ```

<br/>

- `#`은 문서 최상위 제목에만 사용합니다.
- 계층을 건너뛸 수는 없습니다.

> ```js
> # -> ###: X
> ```

<br/>

숫자 제목을 포함할 경우에는 하위 계층에 상위 계층의 수를 `-`(하이픈)으로 연결하여 작성합니다.

> ```js
> # 1. 제목
> ## 1-1. 부제목
> ```

# 2. 목록 구조
## 2-1. 순서가 없는 목록의 경우
모든 목록(`<li>`)에 순서가 없는 항목을 나열할 경우, `-`(하이픈)을 사용하여 작성합니다.

> ```js
> - 항목
> ```

<br/>

- 들여쓰기는 특별한 이유가 없는 경우, 공백 `2`칸을 사용합니다.

> ```js
> - 항목
>   - 부항목
>     - 소항목
> ```

## 2-2. 순서가 있는 목록의 경우
모든 목록(`<li>`)에 순서가 있는 항목을 나열할 경우, `n.`(숫자 + `.`)을 사용합니다.

> ```js
> 1. 첫 번째
> 2. 두 번째
> ```

<br/>

- 들여쓰기는 특별한 이유가 없는 경우, 공백 `2`칸을 사용합니다.
- 계층적 목록의 구현이 필요한 경우, 하위 계층에 상위 계층의 수를 `-`(하이픈)으로 연결하여 작성합니다.

> ```js
> 1. 첫 번째
>   1-1. 첫 번재 항목의 첫 번째
> ```

# 3. 단어 강조
## 3-1. 일반 본문 강조
기본적인 단어는 `**`을 사용하여 강조합니다.

> ```js
> **강조**
> ```

## 3-2. 특정 용어 강조
위키에서 소개하는 기술 용어 혹은 고유명사는 ``(backtick)를 사용하여 강조합니다.

> ```js
> `datapack`
> `minecraft`
> ```

<br/>

- 괄호 내의 기술 용어나 고유명사는 ``(backtick)을 사용하여 강조하지 **않습니다.**

> ```js
> `minecraft`(마인크래프트): O
> `datapack`(`데이터 팩`): X
> ```

<br/>

## 3-3. 링크 강조
링크는 `[]`(대괄호)를 보이게 강조합니다.

> ```js
> [[`google`](https://google.com)]
> ```
